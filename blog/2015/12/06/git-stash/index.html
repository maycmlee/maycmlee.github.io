Git Stash

We started our second project mode last week. Compared to the first project mode I’m feeling more comfortable with using Github. Here, I just wanted to go over the git stash command because I’ve found it really helpful when I need to put a feature I’m working on on hold to work on something else.

So, for example, say I’m trying to write a new feature, but was asked to fix a bug in another part of the code.

So if I did git status

I get…

I don’t really want to commit my test code yet because it’s only half done. So, instead, I can do git stash which is like a ‘pause button’ for your current work.

By doing git stash you save your work on your local directory and clear out your working directory.

After fixing the bug, to get back the work I was doing before I just have to use git pop, which gives me the most recent work I have stashed.

You can also stash more than one work-in-progress and git the list of stashed by using git stash list

You can see all of your stashes by using git stash list
